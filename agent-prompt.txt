IMPLEMENT ISSUE #382: feat: add playback latency calibration and manual override
## User Story
As a user, I want ACE-Step DAW to detect and expose playback latency automatically, so that timing feels accurate on my device without manual guesswork.

## Source
- `docs/design/UX_IMPROVEMENT_CHECKLIST.md`
- Section: `P0 — Critical (Must Ship) > Core Audio & Transport`
- Checklist item: `Latency calibration`

## Problem
The app does not yet auto-detect output latency from the Web Audio stack or expose a manual correction path. That creates timing drift between what users hear and what they see, especially across different audio devices and browsers.

## Scope
- Detect `audioContext.outputLatency` and `audioContext.baseLatency` on first use when available
- Surface detected latency in settings UI
- Allow manual override when browser-reported values are missing or inaccurate
- Persist the chosen latency value
- Make the value available to playback and visual compensation systems

## Acceptance Criteria
- On first audio-enabled session, the DAW captures available latency values
- Settings displays detected latency in milliseconds
- Users can manually override the value and save it
- Reloading restores the saved override
- Playback systems can consume a single normalized latency setting
- Missing browser support degrades gracefully with a clear fallback state
